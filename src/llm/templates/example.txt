• approximate endpoint paths
• create new endpoints
• partially match endpoints

• infer missing values
• interpolate values
• fabricate defaults
• use optional dependencies (db_controller, api_logger, etc.)

• Matching is case-sensitive.
• No mapping, synonym resolution, or normalization is allowed.

If the user provides an invalid enum value → return INVALID_ENUM_VALUE.

If the enum definition is required but unavailable → return MISSING_REQUIRED_INFORMATION.
